# 0-1 项目的常见问题

---

## 1. 什么是懒加载？为什么要懒加载？

### 懒加载是什么？

**懒加载**（Lazy Loading）就是：**不在程序启动时初始化，而是等到真正需要的时候才初始化。**

### 为什么本项目必须懒加载？

RAG 系统需要加载 4 个 ML 模型：

| 模型 | 用途 | 大小 |
|------|------|------|
| `bert-base-chinese` | 查询分类 | ~400MB |
| `bert_query_classifier` | 微调分类头 | ~400MB |
| `bge-m3` | 向量嵌入（稠密+稀疏） | ~2GB |
| `bge-reranker-large` | 交叉编码器重排序 | ~1.2GB |

**总计约 4GB，每个加载需 5-30 秒。**

### 不用懒加载的后果

Django 的 `manage.py` 是所有命令的统一入口，每个命令都会先启动 Django、加载所有 app：

```
python manage.py runserver     ← 要等 60 秒模型加载完才能启动
python manage.py migrate        ← 只是建个表，也要等 60 秒
python manage.py makemigrations ← 只是生成迁移文件，也要等 60 秒
python manage.py shell          ← 只是想调试一下，也要等 60 秒
```

**更严重的问题——数据库迁移死锁：**

如果在 `apps.py` 的 `ready()` 中加载模型：

```python
# 错误做法
class RagConfig(AppConfig):
    def ready(self):
        from rag.core import IntegratedQASystem
        IntegratedQASystem()  # 此时读 FQA 表
```

第一次 `migrate` 时：
1. Django 启动 → 加载 `rag` app → 执行 `ready()`
2. `IntegratedQASystem` 尝试从 MySQL 读 FQA 数据
3. 但 `rag_fqa_entry` 表还不存在 → 崩溃
4. `migrate` 永远无法执行 → **死锁**

### 正确做法：懒加载单例

```python
# rag/core/__init__.py
_qa_system = None  # 一开始是空的

def get_qa_system():
    global _qa_system
    if _qa_system is None:
        # 第一次有人调用时才加载（仅此一次）
        _qa_system = IntegratedQASystem()
    return _qa_system
```

### 效果对比

| | 启动时加载 | 懒加载 |
|---|---|---|
| `migrate` | 等 60s + 可能崩溃 | 秒级 |
| `runserver` | 等 60s | 秒级 |
| 首次问答请求 | 正常 | 等 60s（仅一次） |
| 后续问答请求 | 正常 | 正常（复用单例） |
| 不访问问答功能 | 白白浪费 4GB 内存 | 零开销 |
